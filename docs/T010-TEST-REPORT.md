## 📋 測試範圍

根據方案 A 設計，弱點分析頁面應包含兩個獨立的弱點展示區塊：

### 1. **「最常錯的選項」區塊**（選項層級分析）
- **位置：** 即時成就感模組內（第 375-390 行）
- **功能：** 顯示 Top 3-5 題中，學員最常選錯的「選項」
- **數據來源：** `getWrongAnswerPatterns()` API
- **展示內容：**
  - 題目資訊（科目、章節、題幹）
  - 最常選錯的選項（如「選項 B」）
  - 該選項被選錯的次數

### 2. **「需要加強的題目」區塊**（題目層級列表）
- **位置：** 頁面下半部（第 577-585 行）
- **功能：** 完整的弱點題目列表（最多 30 題）
- **數據來源：** `getWeaknessList()` API
- **展示內容：**
  - 題目卡片（含錯誤率、答錯次數）
  - 特殊標記（最需要搶救、次要警告）
  - 點擊跳轉複習功能

## 📊 用戶使用流程

```
用戶進入弱點分析頁
    ↓
看到「考試倒數」（若已設定）
    ↓
看到「即時成就感模組」
    ├─ 今日新增熟練（激勵）
    ├─ 本次表現（自我比較）
    └─ 最常錯的選項（了解錯誤模式） ⭐
          ↓
    點擊「展開全部」查看完整錯誤選項分析
          ↓
    點擊某個錯誤選項，跳轉到該題複習
    ↓
向下滾動，看到「需要加強的題目」 ⭐
    ↓
依錯誤率排序，優先複習「最需要搶救」的題目
    ↓
點擊題目卡片，開始複習
```

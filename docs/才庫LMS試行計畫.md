# 才庫 AI 知識圖卡 LMS — 試行計畫（內部決策文件）

**版本：** v1.0
**日期：** 2026-04-13
**文件用途：** 內部決策，評估是否正式推進、定價策略與商業模式設計

## 一、試行目的

這份計畫的核心問題只有一個：

> **企業客戶願意為這套 LMS 服務付多少錢？**

我們已經驗證了產品可以做出來（MVP 已上線）。接下來要驗證的不是功能，是**商業模型的可行性**——具體說，就是定價。

試行結束時，我們應該能回答：

- 企業客戶對哪個定價方案有感？哪個有阻力？
- 什麼服務組合讓客戶覺得「值這個錢」？
- 才庫作為培訓服務商，LMS 是獨立產品賣、還是打包進服務費更好走？

---

## 二、背景與現況

### 2.1 才庫的角色定位

才庫不是純軟體公司，是**培訓服務商**。LMS 是才庫提供培訓服務的工具之一，不是獨立產品線。這個定位決定了試行策略：

- 我們不是要「賣一套軟體」
- 我們是要驗證「把 LMS 融入才庫培訓服務後，整體方案的定價空間在哪裡」

### 2.2 MVP 當前狀態

| 功能         | 狀態          | 說明                                       |
| ------------ | ------------- | ------------------------------------------ |
| AI 自動建卡  | ✅ 可用       | 丟 PDF 進去，AI 產出知識卡片               |
| 遊戲化學習   | ✅ 可用       | 翻卡、答題、升等、徽章、排行榜             |
| AI 自動出題  | ⚠️ 手動流程 | 功能存在，但尚未整合進系統介面，需人工輔助 |
| 課後行動追蹤 | ❌ 尚未開發   | 列為試行期間觀察需求用                     |

### 2.3 現有成果（才庫內部使用回饋）

才庫事業群已將本系統導入內部知識學習流程，以下為部分成員使用後的真實心得：

> 「以前讀講義就是把 PDF 存在手機裡，然後就沒有然後了。用翻卡的方式一張一張看，不知道為什麼就真的會想翻完。而且翻完還有小測驗，當下就知道自己哪裡沒記住，比自己讀有效多了。」
> — 才庫業務同仁

> 「我沒想到這種東西會讓我有想繼續的感覺。看到自己升級、排名往前，會想說再多做幾題。感覺不像在讀書，比較像在玩。」
> — 才庫培訓組成員

> 「之前辦培訓，講師講完就結束了，完全不知道大家有沒有記住。現在可以看到誰做了測驗、錯在哪裡，主管終於有東西可以追。」
> — 才庫內部主管

*⚠️ 以上為示例文字，正式對外使用前請替換為真實學員回饋。*

## 三、試行設計

### 3.1 試行期間

**2026 年 Q2—Q3，共 3 個月**

| 月份      | 階段   | 核心任務                                        |
| --------- | ------ | ----------------------------------------------- |
| 第 1 個月 | 導入期 | 找到 1-2 家試行企業，完成上線，觀察初期使用狀況 |
| 第 2 個月 | 運行期 | 收集使用數據，中期訪談，測試定價反應          |
| 第 3 個月 | 評估期 | 深度訪談，整理數據，做出定價決策                |

### 3.2 試行對象條件

**理想的試行企業應符合以下條件（2-3 家）：**

| 條件                       | 說明               | 為什麼重要                       |
| -------------------------- | ------------------ | -------------------------------- |
| 才庫現有客戶或熱絡潛在客戶 | 已有信任基礎       | 降低導入阻力，加快決策速度       |
| 員工規模 30-200 人         | 中型企業           | 太小沒有培訓預算，太大決策鏈太長 |
| 有定期內訓需求             | 每季至少辦一次培訓 | 有明確的問題要解決               |
| 有 HR 或培訓承辦人         | 有明確的系統使用者 | 方便收集使用者回饋               |
| 老闆或 HR 主管願意訪談     | 決策者配合度高     | 定價驗證需要問到決策者           |

**優先找的企業類型：** 製造業、金融業、零售連鎖——這三類有明確的員工培訓合規需求，痛點最清晰。

### 3.3 試行條件設計（定價假設）

這是整個試行的核心設計。我們要用「試行條件」來測試市場的價格敏感度。

**建議方案：分層試行**

**方案 A：培訓服務打包（主推）**

> 將 LMS 融入才庫的培訓服務報價中，不單獨提 LMS 費用。

| 項目     | 做法                                                              |
| -------- | ----------------------------------------------------------------- |
| 試行條件 | 才庫提供一套完整培訓方案，LMS 是其中一環，試行期間整體方案打 7 折 |
| 目的     | 測試「LMS 作為服務加值」的定價模式                                |
| 衡量點   | 客戶是否在意 LMS 的存在？還是只在意培訓結果？                     |

**方案 B：LMS 獨立試用（次選）**

> 對願意單獨評估 LMS 的企業，提供試行報價。**試行本身收費，不提供免費試用。**

| 方案名稱 | 試行價格                                          | 正式定價（標準）                      | 試行條件 |
| -------- | ------------------------------------------------- | ------------------------------------- | -------- |
| 入門版   | NT$50,000（試行費）               | NT$100,000/年 | 試行費可折抵第一年年費；試行期 3 個月 |          |
| 專業版   | NT$100,000（試行費） | NT$150,000/年              | 試行費可折抵第一年年費；試行期 3 個月 |          |

> **為什麼不做免費試行：**
> 免費試行驗證的是「有人願意用」，不是「有人願意付錢」。這兩件事完全不同。
> 只有對方掏出錢，才算真正驗證了定價假設。
>
> **折扣空間：** 標準定價 NT$100,000/年，業務視情況最低可到 NT$80,000，但對外報價一律從 10 萬出發。折扣是談判籌碼，不是標價。
>
> **注意：** 方案 B 適合用來單獨測試定價，但可能影響才庫的服務商定位。兩個方案可同時進行於不同客戶，最終比較哪個成交率更高、客戶滿意度更好。

## 四、驗證指標（定價驗證為主）

### 4.1 定價驗證指標（最重要）

| 指標                | 衡量方式                                       | 目標                      |
| ------------------- | ---------------------------------------------- | ------------------------- |
| 客戶願付意願（WTP） | 深度訪談：「如果正式收費，你覺得合理價格是？」 | 取得 3 家以上明確報價意願 |
| 定價阻力點          | 訪談：「哪個價格讓你覺得太貴？」               | 找到清楚的阻力區間        |
| 決策鏈長度          | 觀察：從提案到決策需要幾個層級、多少天         | 了解銷售週期              |
| 試行後轉正意願      | 直接詢問：試行結束後願意付費續用？             | ≥ 1 家明確表示願意付費   |
| 打包 vs 獨立偏好    | 訪談：偏好 LMS 打包進服務費，還是獨立計費？    | 得出明確傾向              |

### 4.2 產品驗證指標（次要，用來支撐定價判斷）

| 指標              | 衡量方式                         | 目標                        |
| ----------------- | -------------------------------- | --------------------------- |
| 學員系統開啟率    | 後台數據：每週登入人數/總人數    | ≥ 50%（基礎活躍）          |
| 知識卡片完成率    | 後台數據：完成全套卡片的學員比例 | ≥ 60%                      |
| HR 主動操作次數   | 訪談：HR 每週主動進後台幾次？    | 至少每週 1 次（代表有在用） |
| 主管對 LMS 的評分 | 試行結束問卷：1-10 分            | ≥ 7 分                     |

> **說明：** 產品指標的目的不是衡量產品好不好，而是：如果數據太差，表示客戶根本沒在用，這時候問出來的定價意願是沒有參考價值的。所以產品指標是定價驗證的**前提條件**。

## 六、執行成本估算

| 項目                 | 成本             | 說明                     |
| -------------------- | ---------------- | ------------------------ |
| Supabase（每家客戶） | NT$0-800/月      | Free tier 可撐前期       |
| AI 建卡 API 費用     | ~NT$500-1,500/家 | 依知識卡片數量           |
| 人力：導入與陪跑     | 約 20-30 小時/家 | 含上線、訓練、中期支援   |
| 人力：訪談與整理     | 約 10-15 小時    | 含訪談、逐字稿、分析報告 |

**3 個月試行總成本估算（2 家試行企業）：**

- 直接成本：約 NT$5,000-10,000（API + Supabase）
- 機會成本：約 60-90 小時人力

> 這是一個**低成本、高資訊密度**的試行設計——花不到 10 萬，換到定價策略的明確答案。

## 七、試行結束後的決策框架

試行結束時，根據數據回答以下三個問題，做出決策：

### 問題一：客戶願意付費嗎？

| 結果                                      | 決策                                                    |
| ----------------------------------------- | ------------------------------------------------------- |
| ≥ 1 家明確願意付費，且報價與我們預期相近 | **繼續推進，正式定價**                            |
| 願意付費，但報價遠低於預期                | **重新評估服務組合，降成本或調整功能範圍**        |
| 沒有人願意付費                            | **停止 LMS 獨立收費，改為才庫培訓服務的隱性加值** |

### 問題二：打包賣還是獨立賣？

| 結果                              | 決策                                       |
| --------------------------------- | ------------------------------------------ |
| 客戶不在意 LMS，只在意培訓結果    | **LMS 打包進才庫服務費，不單獨計價** |
| 客戶主動問 LMS 的功能，對系統有感 | **LMS 可以獨立計費，但保留打包選項** |
| 客戶想要 LMS 但不想要才庫培訓     | **評估是否要開放純 LMS 訂閱路線**    |

### 問題三：定價區間在哪裡？

根據訪談中客戶的反應，標記出以下三個價格點：

- **太便宜（懷疑品質）**：低於 ______ 元
- **合理區間（願意付）**：______ 至 ______ 元
- **太貴（產生阻力）**：超過 ______ 元

> 這三個空格是試行最重要的產出。試行結束後填入，作為正式定價依據。

## 八、風險與應對

| 風險                       | 可能性 | 應對方式                                                         |
| -------------------------- | ------ | ---------------------------------------------------------------- |
| 找不到試行企業             | 中     | 降低條件，才庫內部部門先當白老鼠；或從才庫既有客戶中找最熟的一家 |
| 企業上線後沒人用           | 中     | 第一個月安排 1-2 次線上說明會，HR 陪學員一起用                   |
| AI 出題手動流程拖慢導入    | 高     | 試行期間由才庫團隊代勞出題，不讓客戶感受到限制                   |
| 客戶覺得功能不夠、不願付費 | 低-中  | 先問「缺什麼」，把答案記下來，這比任何 A/B Test 都有價值         |
| 試行期間系統出問題         | 低     | 才庫已有 360d 自用的穩定環境，技術風險可控                       |

## 九、附錄：訪談問題清單

**給 HR 主管 / 培訓承辦人：**

1. 現在的培訓流程中，哪一步最讓你頭痛？
2. 用了這套系統之後，哪個功能對你最有幫助？哪個你根本不用？
3. 如果這套系統正式收費，你覺得合理的價格是？（開放式，讓對方先說）
4. 這套費用你要自己決定，還是要上呈？上呈需要什麼樣的數據？
5. 如果才庫把 LMS 打包進培訓方案，整體費用比現在高 20-30%，你能接受嗎？

**給老闆 / 決策者：**

1. 你認為員工培訓的最大問題是什麼？這套系統有解決嗎？
2. 如果系統能讓你看到「哪些人學了、學了什麼、有沒有用出來」，這對你來說值多少錢？
3. 你現在一年花多少在員工培訓上？（講師費 + 場地 + 時間成本）
4. 什麼數據能讓你覺得這筆錢花得值？

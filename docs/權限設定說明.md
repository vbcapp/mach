# 權限設定說明

## 角色總覽

系統共有 4 種角色，儲存在資料庫 `users.role` 欄位：

| 角色值 | 中文名稱 | 說明 |
|--------|----------|------|
| `master_admin` | 最高管理者 | 系統擁有者，擁有所有權限 |
| `super_admin` | 高級管理員 | 與最高管理者相同權限（目前未使用） |
| `sub_admin` | 次級管理員 | 一般管理員，可匯入題目與管理系統 |
| `user` | 一般用戶 | 學員，只能做題與查看個人資料 |

> **備註：** `super_admin` 目前沒有指派給任何帳號，實際只用了 `master_admin`、`sub_admin`、`user` 三種。

---

## 權限對照表

| 功能 | `master_admin` | `super_admin` | `sub_admin` | `user` |
|------|:-:|:-:|:-:|:-:|
| 做題 / 查看成績 | O | O | O | O |
| 個人資料頁 (`profile.html`) | O | O | O | O |
| 管理員儀表板（數據概覽） | O | O | O | X |
| 匯入題目 (`import.html`) | O | O | O | X |
| 手動建立題目 (`create.html`) | O | O | O | X |
| 權限管理 (`admin.html`) | O | O | O | X |

> 目前所有管理員角色的功能權限相同，皆使用 `RoleManager.isAdmin()` 判斷。

---

## 權限判斷函式

定義於 [config.js](config.js) 的 `RoleManager` 物件：

| 函式 | 匹配的角色 | 用途 |
|------|-----------|------|
| `isAdmin()` | `master_admin`、`super_admin`、`sub_admin` | 判斷是否為任何管理員 |
| `isSuperAdmin()` | `master_admin`、`super_admin` | 判斷是否為高級管理員以上 |
| `isMasterAdmin()` | `master_admin` | 判斷是否為最高管理者 |
| `isSubAdmin()` | `sub_admin` | 判斷是否為次級管理員 |

### 使用方式

```javascript
// 初始化（通常在頁面載入時由 apiService 自動呼叫）
await RoleManager.init(supabaseClient);

// 判斷權限
RoleManager.isAdmin()       // 任何管理員
RoleManager.isSuperAdmin()  // 高級管理員以上
RoleManager.isMasterAdmin() // 最高管理者
RoleManager.isSubAdmin()    // 次級管理員

// 取得角色字串
RoleManager.role            // 'master_admin' | 'super_admin' | 'sub_admin' | 'user'

// 取得最高管理者的 user ID
RoleManager.masterAdminId
```

## 各頁面的權限檢查

| 頁面 | 檢查方式 | 不符合時的行為 |
|------|---------|---------------|
| `profile.html` | `RoleManager.isAdmin()` | 管理員儀表板及匯入區塊隱藏，僅顯示學員功能 |
| `import.html` | `RoleManager.isAdmin()` | 彈出「權限不足」提示，導回 `profile.html` |
| `create.html` | `RoleManager.isAdmin()` | 彈出「權限不足」提示，導回 `profile.html` |
| `admin.html` | `RoleManager.isAdmin()` | 彈出「權限不足」提示，導回 `profile.html` |
| `index.html` | `RoleManager.isAdmin()` | 隱藏管理員專用的 UI 元素 |

## 如何修改用戶角色

直接在 Supabase Dashboard 修改 `users` 表的 `role` 欄位：

1. 進入 Supabase Dashboard → Table Editor → `users`
2. 找到目標用戶
3. 將 `role` 欄位改為以下其中一個值：
   - `master_admin`
   - `super_admin`
   - `sub_admin`
   - `user`
4. 儲存後，用戶重新登入即生效

> **注意：** `role` 欄位有 CHECK 約束，只接受上述 4 個值，輸入其他值會報錯。

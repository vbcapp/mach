# 切換客戶步驟（SaaS 部署指南）

## 架構說明

每個企業客戶 = 一個 GitHub 專案 + 一個 Supabase 專案。
程式碼相同，只有 `config.js` 的 2 個值不同。
**企業名稱、Logo、說明文字等品牌資訊全部從資料庫 `organization_settings` 表自動讀取**，改資料庫就會自動生效。

## Step 1：建立新的 Supabase 專案

1. 到 [Supabase Dashboard](https://supabase.com/dashboard) 建立新專案
2. 區域建議選 `ap-northeast-1`（東京，離台灣最近）
3. 記下專案的 **URL** 和 **anon key**（在 Settings → API 可找到）

## Step 2：套用資料庫 Schema

在新專案的 SQL Editor 執行以下步驟：

1. 執行 `database.md` 中的所有建表 SQL（users、questions、user_question_progress、answer_records、random_test_sessions、chapter_access、tags、user_ai_analyses、user_badges、user_records 等）
2. 執行 `sql/` 資料夾內的所有 RPC 函數：
   - `admin_dashboard_functions.sql`
   - `admin_attempt_accuracy.sql`
   - `get_user_daily_streak.sql`
3. 用 Claude Code + Supabase MCP 建立 `organization_settings` 表並插入該企業的品牌資料（最簡單的方式）：

**方法 A：用 Claude Code（推薦）**

直接跟 Claude 說：
> 請在 Supabase 專案 `xxx` 建立 organization_settings 表，企業資訊如下：
> - org_name: ○○企業名稱
> - short_name: ○○簡稱
> - description: ○○企業證照考試練習平台
> - footer_text: ○○企業名稱 Flashcard App v1.2.0
> - logo_url: https://xxx.com/logo.png

Claude 會自動透過 Supabase MCP 建表、設定 RLS、插入資料。

**方法 B：手動在 SQL Editor 執行**

```sql
CREATE TABLE IF NOT EXISTS public.organization_settings (
    id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
    org_name text NOT NULL,
    short_name text NOT NULL,
    description text DEFAULT '',
    footer_text text DEFAULT '',
    app_version text DEFAULT 'v1.0.0',
    primary_color text DEFAULT '#FFD600',
    logo_url text DEFAULT '',
    created_at timestamptz DEFAULT now(),
    updated_at timestamptz DEFAULT now()
);

ALTER TABLE public.organization_settings ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow public read" ON public.organization_settings FOR SELECT USING (true);

-- ⚠️ RLS 更新權限：根據 users 表是否有 role 欄位選擇其一
-- 如果 users 表有 role 欄位：
CREATE POLICY "Allow admin update" ON public.organization_settings FOR UPDATE USING (
    EXISTS (SELECT 1 FROM public.users WHERE id = auth.uid() AND role IN ('master_admin', 'super_admin'))
);
-- 如果 users 表沒有 role 欄位（舊版資料庫）：
-- CREATE POLICY "Allow authenticated update" ON public.organization_settings FOR UPDATE USING (auth.role() = 'authenticated');

-- ⬇️ 改成新企業的資料
INSERT INTO public.organization_settings (org_name, short_name, description, footer_text, app_version, primary_color, logo_url)
VALUES (
    '○○企業名稱',
    '○○簡稱',
    '○○企業證照考試練習平台',
    '○○企業名稱 Flashcard App v1.2.0',
    'v1.2.0',
    '#FFD600',
    'https://xxx.com/logo.png'   -- Logo 圖片網址（可留空）
);
```

4. 設定 RLS 政策（確保所有表都有啟用）
5. 在 `users` 表手動建立一個 `master_admin` 帳號（如果 users 表有 role 欄位）

## Step 3：Fork GitHub 專案 & 修改 config.js

1. Fork 主專案到新的 GitHub repo
2. 打開 `config.js`，修改最上方的 **2 個值**：

```javascript
const SUPABASE_CONFIG = {
    url: 'https://○○○○○○○○.supabase.co',   // ← 新專案的 URL
    anonKey: '○○○○○○○○'                     // ← 新專案的 anon key
};
```

**只改這 2 行，其他檔案完全不用動。**
企業名稱、Logo 等品牌資訊會自動從資料庫的 `organization_settings` 表讀取（由 `config.js` 的 `OrgBranding.fetch()` 處理）。

## Step 4：部署

用 Zeabur、GitHub Pages、Netlify、Vercel 或任何靜態主機部署即可。

## Step 5：設定 Supabase Auth（如果有用 Google 登入）

到新 Supabase 專案 → Authentication → URL Configuration：
- Site URL 設為新網站的網址（例如 `https://xxx.zeabur.app`）
- Redirect URLs 加上 `https://xxx.zeabur.app/**`

## 之後要改企業資訊？

**方法 A：用 Claude Code（最快）**
> 請把 Supabase 專案 `xxx` 的 organization_settings 的 org_name 改成「新名稱」

**方法 B：到 Supabase Dashboard**
直接修改 `organization_settings` 表即可，不用動程式碼：

| 欄位 | 用途 | 範例 |
|------|------|------|
| `org_name` | 企業全名（頁首、登入頁、頁面標題） | 工商安全衛生協會-新竹職訓中心 |
| `short_name` | 簡稱（PWA 圖示下方文字） | 職訓考試 |
| `description` | 說明文字 | 企業證照考試練習平台 |
| `footer_text` | 頁尾文字 | 工商安全衛生協會 Flashcard App v1.2.0 |
| `app_version` | 版本號 | v1.2.0 |
| `logo_url` | Logo 圖片網址（通常填部署網址） | https://sandys-exam-card.zeabur.app/ |
| `primary_color` | 品牌主色 | #FFD600 |

改完存檔，重新整理網頁就會看到新的資訊。
